1. Pengenalan

Tentang T4n OS

T4n OS adalah distribusi Linux berbasis Void Linux yang dioptimalkan untuk kinerja tinggi dan pengalaman pengguna yang mulus. Didesain untuk pengguna yang menginginkan sistem yang ringan, cepat, dan dapat disesuaikan.

T4n OS menggabungkan kestabilan Void Linux dengan berbagai peningkatan dan tweak khusus untuk memberikan pengalaman computing yang optimal. Sistem ini menggunakan paket manajer XBPS yang cepat dan andal, serta mendukung berbagai lingkungan desktop modern.

T4n OS berasal dari nama Tokoh revolusioner di Indonesia yaitu Tan Malaka

Catatan: T4n OS adalah proyek independen dan tidak berafiliasi dengan Void Linux secara resmi.

2. Edisi T4n OS

T4n OS tersedia dalam beberapa edisi yang dioptimalkan untuk berbagai kebutuhan dan preferensi pengguna:

XFCE Edition

Edisi XFCE menawarkan lingkungan desktop yang ringan, stabil, dan mudah digunakan. Cocok untuk pengguna yang menginginkan keseimbangan antara kinerja dan fitur.

i3 Edition

Edisi i3 menggunakan window manager tiling yang efisien untuk produktivitas maksimal. Ideal untuk pengguna yang lebih suka keyboard-driven workflow.

BSPWM Edition

Edisi BSPWM menawarkan tiling window manager yang ringan dan dapat dikonfigurasi tinggi dengan fokus pada kinerja dan minimalisme.

Hyprland Edition

Edisi Hyprland menggunakan compositor Wayland yang dinamis dengan animasi halus dan fitur modern. Cocok untuk pengguna yang menginginkan pengalaman visual yang menarik.

3. Persiapan Instalasi

Sebelum memulai instalasi T4n OS, pastikan Anda telah memenuhi persyaratan berikut:

  • Hardware: Prosesor 64-bit (x86_64), minimal 2GB RAM (4GB direkomendasikan), dan minimal 15GB ruang disk
  • Media Instalasi: USB flash drive minimal 4GB
  • Koneksi Internet: Diperlukan untuk mengunduh paket selama instalasi
  • Backup Data: Pastikan Anda telah membackup data penting sebelum instalasi

Unduh ISO T4n OS

Unduh file ISO T4n OS dari situs resmi:

https://tanos.org/download/

Verifikasi ISO

Setelah mengunduh, verifikasi integritas file ISO menggunakan checksum:

sha256sum t4n-os-*.iso

4. Pemasangan T4n OS

Proses instalasi T4n OS dirancang untuk sederhana namun fleksibel. Ikuti langkah-langkah berikut untuk menginstal sistem:

1. Boot dari Live USB

Masukkan USB flash drive yang berisi ISO T4n OS dan boot dari media tersebut. Pilih opsi "Boot T4n OS Live" dari menu boot.

2. Persiapkan Partisi

Gunakan alat partisi seperti fdisk, cfdisk, atau gparted untuk mempersiapkan partisi:

cfdisk /dev/sda

3. Mount Partisi

Mount partisi root dan home (jika ada):

mount /dev/sda2 /mnt
mkdir -p /mnt/home
mount /dev/sda3 /mnt/home

4. Instal Sistem Dasar

Jalankan instalasi dengan perintah:

t4n-installer

5. Konfigurasi Sistem

Setelah instalasi selesai, konfigurasi sistem dasar seperti pengguna, jaringan, dan driver.

6. Reboot

Keluar dari lingkungan chroot dan reboot sistem:

exit
umount -R /mnt
reboot

4.1 Live USB

Untuk membuat Live USB T4n OS, Anda dapat menggunakan salah satu metode berikut:

Menggunakan dd (Linux/macOS)

dd if=t4n-os-*.iso of=/dev/sdX bs=4M status=progress

Ganti /dev/sdX dengan device USB Anda (misal: /dev/sdb).

Menggunakan Etcher (Cross-platform)

  1. Unduh dan instal Etcher dari https://etcher.balena.io/
  2. Pilih file ISO T4n OS
  3. Pilih USB flash drive target
  4. Klik "Flash" untuk memulai proses

Menggunakan Rufus (Windows)

  1. Unduh dan instal Rufus dari https://rufus.ie/
  2. Pilih USB flash drive target
  3. Pilih opsi "DD Image" dan pilih file ISO T4n OS
  4. Klik "Start" untuk memulai proses

Peringatan: Proses ini akan menghapus semua data di USB flash drive. Pastikan Anda telah membackup data penting sebelum melanjutkan.

4.1.1 Akun Pengguna

Saat membuat Live USB, Anda perlu mengkonfigurasi akun pengguna untuk lingkungan live:

Membuat Akun Pengguna Live

# Buat user live
useradd -m -G wheel,audio,video,storage t4n
echo "t4n:password" | chpasswd

Mengatur Password Root

echo "root:password" | chpasswd

Konfigurasi Sudo

Untuk memungkinkan user t4n menggunakan sudo:

echo "%wheel ALL=(ALL:ALL) ALL" > /etc/sudoers.d/wheel

Keamanan: Password default di lingkungan live sebaiknya diganti setelah boot untuk keamanan.

4.1.2 Boot Mode

T4n OS mendukung dua mode boot: BIOS (Legacy) dan UEFI. Pastikan Anda memilih mode yang sesuai dengan sistem Anda:

Memeriksa Mode Boot

# Untuk UEFI
ls /sys/firmware/efi

# Jika direktori ada, sistem menggunakan UEFI

Boot Mode untuk Live USB

  • UEFI Mode: Pilih opsi boot dengan label "UEFI" di menu boot
  • Legacy Mode: Pilih opsi boot tanpa label "UEFI"

Implikasi Boot Mode

Mode boot yang Anda pilih akan menentukan:

  • Tabel partisi (GPT untuk UEFI, MBR untuk Legacy)
  • Lokasi bootloader (EFI partition untuk UEFI, MBR untuk Legacy)
  • Driver yang digunakan saat boot

Tip: Sebagian besar komputer modern (produksi setelah 2010) mendukung UEFI. Gunakan UEFI jika memungkinkan untuk fitur keamanan yang lebih baik.

4.2 Sistem Partisi

Partisi yang tepat sangat penting untuk kinerja dan stabilitas sistem. Berikut adalah skema partisi yang direkomendasikan untuk T4n OS:

Tipe Disk

  • Root (/): Minimal 15GB, disarankan 25-50GB untuk instalasi yang lebih lengkap
  • Home (/home): Sisa ruang disk untuk data pengguna
  • Swap: Minimal 2GB atau sama dengan ukuran RAM jika hibernasi diperlukan

Contoh Skema Partisi

/dev/sda1 - EFI System Partition (512MB, FAT32)
/dev/sda2 - Root Partition (25GB, ext4)
/dev/sda3 - Home Partition (Sisa ruang, ext4)
/dev/sda4 - Swap Partition (4GB, linux-swap)

Membuat Partisi dengan cfdisk

cfdisk /dev/sda

Dalam cfdisk:

  1. Pilih "gpt" untuk tabel partisi GPT (modern) atau "dos" untuk MBR (legacy)
  2. Buat partisi baru dengan "New"
  3. Tetapkan ukuran dan tipe partisi
  4. Tulis perubahan dengan "Write"
  5. Keluar dengan "Quit"

Format Partisi

Setelah partisi dibuat, format partisi tersebut:

mkfs.fat -F32 /dev/sda1 # EFI partition
mkfs.ext4 /dev/sda2 # Root partition
mkfs.ext4 /dev/sda3 # Home partition
mkswap /dev/sda4 # Swap partition
swapon /dev/sda4 # Activate swap

4.2.1 Tipe Disk (Root & User)

Pemilihan tipe disk yang tepat untuk partisi root dan user sangat penting untuk kinerja sistem:

Partisi Root (/)

Partisi root berisi sistem operasi dan aplikasi:

  • Ukuran minimal: 15GB
  • Ukuran disarankan: 25-50GB
  • Filesystem: ext4 (default), btrfs, atau f2fs untuk SSD
  • Mount options: defaults,noatime untuk performa lebih baik

Partisi Home (/home)

Partisi home berisi data pengguna:

  • Ukuran: Sisa ruang disk yang tersedia
  • Filesystem: ext4, btrfs (untuk snapshot), atau xfs
  • Mount options: defaults,noatime
  • Separate partition: Memudahkan reinstall tanpa kehilangan data

Format Partisi

# Format root partition
mkfs.ext4 -L ROOT /dev/sda2

# Format home partition
mkfs.ext4 -L HOME /dev/sda3

Mount Partisi

# Mount root
mount /dev/sda2 /mnt

# Buat dan mount home
mkdir /mnt/home
mount /dev/sda3 /mnt/home

Tip: Menggunakan filesystem yang sama untuk root dan home (misal ext4) menyederhanakan manajemen dan kompatibilitas.

4.2.2 Swap

Swap adalah ruang di disk yang digunakan ketika RAM penuh. Berikut adalah panduan untuk mengkonfigurasi swap:

Ukuran Swap yang Direkomendasikan

  • RAM < 2GB: Swap = 2x RAM
  • RAM 2-8GB: Swap = RAM
  • RAM > 8GB: Swap = minimal 4GB atau 8GB
  • Untuk hibernasi: Swap = RAM + sedikit buffer

Membuat Swap Partition

# Buat partition swap (misal /dev/sda4)
# Format sebagai swap
mkswap /dev/sda4

# Aktifkan swap
swapon /dev/sda4

Menggunakan Swap File

Alternatif untuk swap partition adalah swap file:

# Buat swap file 4GB
fallocate -l 4G /swapfile

# Atur permission
chmod 600 /swapfile

# Format sebagai swap
mkswap /swapfile

# Aktifkan swap
swapon /swapfile

Konfigurasi Swappiness

Swappiness mengontrol seberapa sering sistem menggunakan swap:

# Lihat nilai swappiness saat ini
cat /proc/sys/vm/swappiness

# Set swappiness ke 10 (lebih sedikit menggunakan swap)
echo "vm.swappiness=10" >> /etc/sysctl.d/99-swappiness.conf

Performa: Untuk sistem dengan RAM besar (>8GB), swap dapat dikurangi atau bahkan dihilangkan kecuali untuk hibernasi.

5. Konfigurasi Sistem

Setelah instalasi dasar selesai, Anda perlu mengkonfigurasi sistem untuk penggunaan optimal:

1. Konfigurasi Pengguna

Buat pengguna baru dan atur password:

useradd -m -G wheel,audio,video,storage username
passwd username

2. Konfigurasi Sudo

Edit file sudoers untuk memberikan akses sudo kepada grup wheel:

visudo

Hapus komentar pada baris: %wheel ALL=(ALL:ALL) ALL

3. Konfigurasi Jaringan

Untuk koneksi nirkabel, instal dan konfigurasi NetworkManager:

xbps-install -S NetworkManager
ln -s /etc/sv/NetworkManager /var/service/
sv up NetworkManager

4. Konfigurasi Locale

Atur locale untuk bahasa Indonesia:

echo "id_ID.UTF-8 UTF-8" >> /etc/locale.conf
localectl set-locale LANG=id_ID.UTF-8

5. Konfigurasi Timezone

Atur timezone ke Asia/Jakarta:

ln -sf /usr/share/zoneinfo/Asia/Jakarta /etc/localtime
hwclock --systohc

5.1 Root

Akses root di T4n OS dapat dikonfigurasi dengan beberapa cara:

Menggunakan Sudo

Cara yang direkomendasikan adalah menggunakan sudo untuk akses administratif:

sudo

Mengaktifkan Login Root Langsung

Jika Anda memerlukan login root langsung (tidak disarankan untuk keamanan):

passwd root

Mengunci Akun Root

Untuk meningkatkan keamanan, kunci akun root dan gunakan sudo saja:

passwd -l root

Keamanan: Menggunakan sudo lebih aman daripada login root langsung karena memberikan kontrol yang lebih baik atas akses administratif.

5.2 Driver

T4n OS mendukung berbagai driver hardware. Berikut adalah panduan untuk menginstal driver yang sesuai:

Driver Intel

Untuk grafis Intel:

xbps-install -S intel-media-driver intel-vaapi-driver
# Untuk Xorg
xbps-install -S xf86-video-intel

Driver AMD

Untuk grafis AMD:

xbps-install -S mesa-vulkan-radeon mesa-dri
# Untuk Xorg

Driver NVIDIA

Untuk grafis NVIDIA:

# Driver open-source (nouveau)

# Driver proprietary (disarankan untuk performa maksimal)
xbps-install -S nvidia

Catatan: Setelah menginstal driver NVIDIA, Anda mungkin perlu mengkonfigurasi Xorg atau Wayland untuk menggunakan driver tersebut.

5.2.1 Driver Intel

Driver Intel untuk grafis dan media di T4n OS:

Driver Grafis Intel

# Instal driver grafis Intel

Driver Xorg untuk Intel

# Untuk Xorg

Driver VA-API untuk Intel

# Untuk akselerasi video

Konfigurasi Xorg untuk Intel

Buat file konfigurasi Xorg untuk Intel:

mkdir -p /etc/X11/xorg.conf.d
echo 'Section "Device"
  Identifier "Intel Graphics"
  Driver "intel"
  Option "TearFree" "true"
EndSection' > /etc/X11/xorg.conf.d/20-intel.conf

Driver untuk Hardware Intel Lainnya

# Driver WiFi Intel
xbps-install -S iwlwifi-firmware

# Driver audio Intel
xbps-install -S alsa-firmware

Driver NVIDIA Proprietary

# Driver open source (basic functionality)

# Untuk komputasi GPU

Rekomendasi: Gunakan driver proprietary NVIDIA untuk performa gaming dan aplikasi grafis intensif.

iptables -P INPUT DROP

6. Grub Bootloader

Konfigurasi GRUB untuk T4n OS:

Instalasi GRUB

# Instal GRUB
xbps-install -S grub

# Untuk UEFI
xbps-install -S grub-x86_64-efi

Konfigurasi GRUB

# Generate konfigurasi GRUB
grub-mkconfig -o /boot/grub/grub.cfg

Instal GRUB ke Disk

# Untuk BIOS/MBR
grub-install /dev/sda

# Untuk UEFI
grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=T4nOS

Konfigurasi Tambahan

Edit file /etc/default/grub untuk penyesuaian:

GRUB_TIMEOUT=5
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
GRUB_THEME="/usr/share/grub/themes/t4n/theme.txt"

6.1 Dual Boot

Konfigurasi dual boot T4n OS dengan Windows atau sistem operasi lain:

Persiapan Partisi

  • Pastikan ada ruang kosong yang cukup untuk T4n OS
  • Gunakan alat partisi Windows untuk mengecilkan partisi yang ada
  • Buat partisi baru untuk T4n OS

Instalasi T4n OS

Ikuti proses instalasi normal, pastikan untuk:

  • Menginstal GRUB ke MBR atau EFI partition
  • Tidak memformat partisi Windows
  • Menggunakan partisi yang sudah disiapkan untuk T4n OS

Konfigurasi GRUB untuk Dual Boot

# Instal os-prober untuk mendeteksi sistem operasi lain
xbps-install -S os-prober

# Generate ulang konfigurasi GRUB
grub-mkconfig -o /boot/grub/grub.cfg

Memperbaiki Bootloader Windows

Jika Windows tidak boot setelah instalasi T4n OS:

# Boot dari media instalasi Windows
# Pilih "Repair your computer"
# Pilih "Troubleshoot" > "Advanced options" > "Command Prompt"
bootrec /fixmbr
bootrec /fixboot
bootrec /rebuildbcd

Peringatan: Selalu backup data penting sebelum mengubah partisi untuk dual boot.

6.2 Reinstall

Panduan untuk menginstal ulang T4n OS:

Backup Data

Sebelum menginstal ulang, backup data penting:

# Backup home directory
tar -czf backup-home.tar.gz /home/username

# Backup konfigurasi penting
tar -czf backup-etc.tar.gz /etc

Proses Reinstall

  1. Boot dari Live USB T4n OS
  2. Mount partisi yang ada
  3. Format hanya partisi root (jaga partisi home jika ada)
  4. Ikuti proses instalasi normal
  5. Restore data dari backup setelah instalasi selesai

Reinstall GRUB

# Boot dari Live USB
# Mount partisi root
mount /dev/sda2 /mnt

# Mount partisi lain jika perlu
mount /dev/sda1 /mnt/boot/efi # untuk UEFI

# Chroot ke sistem
chroot /mnt

# Reinstall GRUB
grub-install /dev/sda
grub-mkconfig -o /boot/grub/grub.cfg

Clean Install

Untuk instalasi bersih (menghapus semua data):

  1. Format semua partisi
  2. Ikuti proses instalasi dari awal
  3. Restore data dari backup eksternal

XBPS Basic Commands

# Update repository
xbps-install -S

xbps-install -u

# Instal paket
xbps-install package-name

xbps-remove package-name

# Cari paket
xbps-query -Rs search-term

Manajemen Repository

# Lihat repository yang dikonfigurasi
xbps-query -L

# Tambahkan repository non-free
xbps-install -S void-repo-nonfree

Informasi Paket

# Info paket

# File yang dimiliki paket
xbps-query -f package-name

# Dependencies
xbps-query -x package-name

7.1 XBPS

XBPS adalah sistem manajemen paket biner untuk Void Linux dan T4n OS:

Update System

# Synchronize repository
xbps-install -S

# Upgrade semua paket
xbps-install -u

# Upgrade system (recommended)
xbps-install -Su

Install Packages

# Install single package
xbps-install package

# Install multiple packages
xbps-install package1 package2 package3

# Install from specific repository
xbps-install -R https://repo.voidlinux.org/current package

Remove Packages

# Remove package
xbps-remove package

# Remove with dependencies
xbps-remove -R package

# Remove orphaned dependencies
xbps-remove -o

# Clean cache
xbps-remove -O

Query Packages

# Search packages
xbps-query -Rs pattern

# Show package information
xbps-query -R package

# List files in package
xbps-query -Rf package

# Show dependencies
xbps-query -Rx package

Hold Packages

# Hold package from updates
xbps-pkgdb -m hold package

# Unhold package
xbps-pkgdb -m unhold package

# Show held packages
xbps-pkgdb -a | grep hold

# Module information
modinfo module-name

Kernel Parameters

Edit /etc/default/grub untuk menambahkan parameter kernel:

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash mitigations=off"

8.1 Kernel Linux LTS

Kernel LTS (Long Term Support) memberikan stabilitas dan dukungan jangka panjang:

Install LTS Kernel

# Install LTS kernel
xbps-install -S linux-lts

# Install LTS headers
xbps-install -S linux-lts-headers

Switch to LTS Kernel

# Set LTS as default in GRUB
# Edit /etc/default/grub
GRUB_DEFAULT="T4n OS GNU/Linux Linux-lts"

Benefits of LTS Kernel

  • Dukungan jangka panjang (biasanya 5+ tahun)
  • Lebih stabil dan teruji
  • Perbaikan keamanan backported
  • Kompatibilitas hardware yang terjamin

Remove Old Kernel

# List installed kernels
xbps-query -s linux

# Remove old kernel
xbps-remove -R linux-version

Rekomendasi: Gunakan kernel LTS untuk server atau sistem yang membutuhkan stabilitas maksimal.

8.2 Kernel Zen

Kernel Zen dioptimalkan untuk performa desktop dan responsivitas:

Install Zen Kernel

# Install Zen kernel
xbps-install -S linux-zen

# Install Zen headers
xbps-install -S linux-zen-headers

Zen Kernel Features

  • Optimasi untuk desktop dan workstation
  • Improved scheduler untuk responsivitas
  • Better gaming performance
  • Audio latency improvements
  • Filesystem optimizations

Zen Kernel Parameters

Tambahkan parameter khusus untuk Zen kernel:

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash elevator=mq-deadline"

Monitoring Performance

# Monitor system responsiveness
sudo perf sched record sleep 10
sudo perf sched latency

Rekomendasi: Kernel Zen ideal untuk desktop, gaming, dan workstation yang membutuhkan responsivitas tinggi.
